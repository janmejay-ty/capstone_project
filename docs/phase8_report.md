# Phase 8: Deployment Readiness (Decoupled API & Frontend) Report

This report documents the decoupled deployment architecture, frontend console UI features, and backend APIs completed in **Phase 8** of the ResolveDesk AI Multi-Agent System.

---

## 1. Decoupled Architecture

The architecture decouples the React SPA client from the FastAPI endpoints:

```
[ React SPA Client ]  <--- CORS (FastAPI: 8000) ---> [ FastAPI Web Server ]
   (Port 5173)                                            (Port 8000)
```

* **FastAPI Backend (`backend/app/main.py`)**: Runs on port 8000, utilizing CORS middleware to accept origins from `http://localhost:5173`.
* **React Frontend SPA (`frontend/`)**: Renders on port 5173 using Vite, TypeScript, and Tailwind CSS v4.

---

## 2. API Endpoints

The backend exposes four core REST API endpoints:

1. **`/api/chat` (POST)**:
   * Accepts a `message` and `session_id`.
   * Queries SQL logs for previous session downvotes to inject adaptive tone notes.
   * Invokes the compiled LangGraph workflow with `MemorySaver` checkpoints matching the thread.
2. **`/api/feedback` (POST)**:
   * Logs thumbs-up/thumbs-down ratings and timestamps directly to the SQLite `feedback_logs` table.
3. **`/api/history/{session_id}` (GET)**:
   * Fetches historical logs for a given session.
4. **`/api/clear/{session_id}` (POST)**:
   * Deletes LangGraph checkpoints for the corresponding session thread:
     ```python
     memory_checkpointer.delete_thread(session_id)
     ```
   * Purges local in-memory trace histories.

## 3. Co-Pilot UI Layout

The React frontend client includes a multi-pane co-pilot workspace:
* **Chat Pane**: Handles message exchanges, bubbles, loading states, and direct thumbs-up/thumbs-down feedback clicks.
* **Agent Trace Panel**:
  * **Current Specialist Node**: Highlights the active worker node (Supervisor, SQL, RAG, Planner, Safety).
  * **Safety Auditing details**: Displays whether security/PII checks passed and if manual escalation is required.
  * **Planner Tasks list**: Displays the sequential step list (from `plan_steps` in state) generated by the Planner Agent.
  * **SQL execution trace**: Renders SQL logs and data returned by database lookups.
* **Theme Selector**: Toggles light/dark styles dynamically.

## 4. Run Launcher Script

A batch script **[run.bat](file:///c:/Users/User/Desktop/python/capstone_project/run.bat)** starts both development servers concurrently:
```batch
@echo off
start cmd /k "cd backend && ..\venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000"
start cmd /k "cd frontend && npm run dev"
```
Both servers are running cleanly in the workspace.
